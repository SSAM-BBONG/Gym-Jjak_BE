# 📚 Trainer Application Changelog

## 2026-07-18 · 내 트레이너 신청 목록·상세 조회 개선 ✨

- 여러 조직에 신청한 트레이너 신청서를 한 화면에서 목록으로 확인할 수 있습니다.
- 기존 `GET /api/trainer-applications/me`은 목록 조회 API로 변경되었습니다.
- 특정 신청서의 상세 내용은 `GET /api/trainer-applications/me/{trainerApplicationId}`로 조회할 수 있습니다.
- 목록에는 헬스장명, 신청 상태, 신청일, 심사일, 반려 사유를 간결하게 제공합니다.
- 본인 소유 신청서만 상세 조회할 수 있어 신청서 정보가 보호됩니다. 🔒

자세한 구현 정책과 검증 내용은 [REVISION.md](REVISION.md)를 참고해주세요. 📚

---

## 2026-07-18 · 다중 조직 신청 생성 API 완료 ✅

- 트레이너 신청 요청이 여러 조직 ID를 받을 수 있게 변경되었습니다.
- 선택한 조직마다 독립된 신청 행이 생성되어, 각 조직이 개별적으로 승인 또는 반려할 수 있습니다.
- 생성 응답은 단일 ID 대신 생성된 신청 ID 목록을 반환합니다.
- 같은 사용자가 같은 조직에 진행 중인 신청을 중복 제출할 수 없도록 DB 제약을 조직 단위로 변경했습니다.
- 수정 API는 단건 신청 ID를 반환하는 별도 응답 객체를 사용합니다.

🔎 설계와 다음 목록 조회 작업은 [REVISION.md](REVISION.md)에서 확인할 수 있습니다.

## 2026-07-18 · 생성 API 후속 보완 ⚙️

- 여러 조직의 활성 상태를 조직 ID별로 반복 확인하던 방식을 `IN` 기반 단일 count 조회로 변경했습니다.
- 요청한 조직 수와 활성 조직 수를 비교해, 하나라도 비활성 또는 존재하지 않으면 저장 전에 요청을 거절합니다.
- V3.3 마이그레이션의 유니크 인덱스명에 맞춰 중복 신청 예외 변환 조건을 갱신했습니다.
- 유니크 충돌은 다시 `DuplicateTrainerApplicationException`으로 변환되어 500 대신 중복 신청 오류로 처리됩니다.
